# RTS-WEB Game Rules & Mechanics (v0.1)

เอกสารนี้รวบรวมกฎเกณฑ์ ระบบการเล่น และโครงสร้างการทำงานของเกม RTS-WEB ทั้งหมด

---

## 1. ระบบการเล่นหลัก (Core Gameplay)
ผู้เล่นจะต้องขยายอาณาจักรโดยการสำรวจพื้นที่ (Blocks), สะสมทรัพยากร, และสร้างกองทัพเพื่อป้องกันฐานหรือโจมตีศัตรู

### ระบบการแพ้-ชนะ (Win/Loss Condition)
*   **เป้าหมายหลัก:** ปกป้อง "ปราสาท (Castle)" ของตนเอง
*   **เมื่อพ่ายแพ้:** หากปราสาทถูกทำลายจน HP เหลือ 0:
    *   ยูนิตทั้งหมดของผู้เล่นคนนั้นจะถูกลบทิ้ง
    *   ทรัพยากรจะถูกรีเซ็ตกลับไปเป็นค่าเริ่มต้น (1,000 G, 1,000 W, 1,000 F)
    *   สิทธิ์การสร้างบล็อกต่อวันจะถูกรีเซ็ต
    *   ผู้เล่นสามารถเริ่มต้นใหม่ได้ทันทีโดยการหาพื้นที่ว่างและสร้างปราสาทหลังใหม่

---

## 2. ทรัพยากร (Resources)
ทรัพยากรมี 3 ประเภทหลัก กระจายตัวอยู่ตามแผนที่ในรูปแบบของทรัพยากรธรรมชาติ:
*   **ทอง (Gold):** ใช้สำหรับสร้างอาคารและผลิตยูนิตทหาร
*   **ไม้ (Wood):** ใช้สำหรับสร้างอาคารและคนเก็บทรัพยากร
*   **อาหาร (Food):** ใช้สำหรับการผลิตยูนิตทุกประเภท
*   **อัตราการเก็บ:** ยูนิต Gatherer จะเก็บทรัพยากรได้ 5 หน่วยต่อรอบการประมวลผล (Tick)

---

## 3. ระบบการสำรวจ (Exploration System)
แผนที่ประกอบด้วย "บล็อก (Blocks)" ขนาด 30x30 ช่อง
*   **การขยายพื้นที่:** ผู้เล่นสามารถสร้างบล็อกใหม่ได้โดยใช้ "โหมดสำรวจ"
*   **เงื่อนไขการสร้าง:** 
    *   ต้องสร้างติดกับบล็อกที่มีอยู่แล้วบนแผนที่
    *   **ค่าใช้จ่าย:** ทอง 200, ไม้ 200, อาหาร 100
    *   **จำกัดจำนวน:** ผู้เล่นแต่ละคนสามารถสร้างบล็อกใหม่ได้เพียง **3 บล็อกต่อวัน**
*   **หมอกแห่งสงคราม (Fog of War):** ยูนิตไม่สามารถเดินเข้าไปในพื้นที่ที่ยังไม่ได้สำรวจได้

## 4. ยูนิตและสิ่งก่อสร้าง (Units & Buildings)

### สิ่งก่อสร้าง (Buildings)
| ชื่อ | HP | ค่าสร้าง (G/W/F) | หน้าที่ |
| :--- | :--- | :--- | :--- |
| **Castle** | 500 | 400/400/200 | ฐานหลัก, ผลิต Gatherer, ป้องกันตัวเองได้ (ระยะ 150) |
| **Tower** | 200 | 200/150/50 | ป้อมป้องกัน, โจมตีอัตโนมัติ (ระยะ 250), ดาเมจสูง |
| **Barracks** | 200 | 150/200/100 | โรงทหาร, ใช้ผลิตยูนิต Soldier |
| **Archery Range** | 200 | 200/250/100 | โรงธนู, ใช้ผลิตยูนิต Archer |

### ยูนิต (Units)
| ชื่อ | HP | พลังโจมตี | ระยะยิง | หน้าที่ |
| :--- | :--- | :--- | :--- | :--- |
| **Gatherer** | 100 | 0 | - | เก็บทรัพยากรธรรมชาติ |
| **Soldier** | 100 | 10 | 30 | ทหารราบโจมตีระยะประชิด |
| **Archer** | 70 | 8 | 150 | ทหารธนูโจมตีระยะไกล |

## 5. การทำงานของโค้ด (Technical Architecture)

### Backend (FastAPI + WebSocket)
*   **Language:** Python
*   **Game Loop:** ทำงานทุกๆ 0.5 วินาที (500ms) เพื่อคำนวณ:
    *   **Movement:** ยูนิตเดินเข้าหาเป้าหมายและหยุดเมื่อถึงระยะ Speed
    *   **Combat:** การตรวจสอบระยะโจมตีระหว่างยูนิต และการยิงอัตโนมัติของป้อม
    *   **Gathering:** การลด HP ของทรัพยากรและเพิ่มทรัพยากรให้ผู้เล่น
*   **State Management:** ข้อมูล World, Blocks, Units และ Players ถูกเก็บไว้ใน Memory (RAM) ของเซิร์ฟเวอร์

### Frontend (Vanilla JS + Canvas)
*   **Language:** JavaScript (ES6)
*   **Rendering:** ใช้ Canvas API วาดภาพ 60 FPS
*   **Data Sync:**
    *   `INIT`: รับข้อมูลแผนที่และยูนิตทั้งหมดเมื่อเข้าเกม
    *   `UNIT_UPDATE`: รับตำแหน่งและสถานะยูนิตแบบ Real-time
    *   `TILE_UPDATE`: รับเฉพาะข้อมูลช่องแผนที่ที่เปลี่ยนแปลง (ประหยัด Bandwidth)
    *   `ATTACK_EVENT`: รับข้อมูลการยิงเพื่อวาดเอฟเฟกต์เลเซอร์

### ระบบเครือข่าย (Communication)
*   ใช้ **WebSocket** เป็นช่องทางหลักในการสื่อสารแบบสองทาง
*   พอร์ตเริ่มต้น: **3000**
